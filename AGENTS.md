# FuseWire-JS - Development Guide

## Tech Stack

- **Runtime:** Node.js >= 18.0.0 (ES modules)
- **Framework:** Express 5
- **Linter:** oxlint
- **Formatter:** oxfmt

## Project Structure

```
fusewire-js/
  docs/                 # Architecture decisions and feature documentation
    architecture.md
  src/
    server.js           # Express application entry point
    components/         # Colocated component files (server + client + html + css)
    static/             # Static files served to the browser
      lib/
        fusewire/       # FuseWire JS component-based framework (client-side)
  test/                 # Test files (*.test.js)
  package.json
```

## Scripts

| Command                 | Description                        |
|-------------------------|------------------------------------|
| `npm start`             | Start the server (`node src/server.js`) |
| `npm test`              | Run tests (`node --test test/**/*.test.js`) |
| `npm run lint`          | Lint source files with oxlint      |
| `npm run format`        | Format source files with oxfmt     |
| `npm run format:check`  | Check formatting without writing   |

## Verification

Before considering a change complete, run:

```bash
npm run lint && npm run format:check && npm test
```

## Constraints

- **Modern JavaScript only.** Use only modern JS features (ES2020+). No legacy patterns, no polyfills.
- **ES modules only.** All files use `import`/`export`. No CommonJS (`require`/`module.exports`).

## Git

- Do NOT add "Generated by" or "Co-Authored-By" lines to commit messages.
- Do NOT `git add` or `git commit` without explicit user approval. Always present the changes first and wait for confirmation.

## Documentation

The `docs/` directory contains architecture decisions and feature documentation. When making changes that affect the system architecture, request flow, or introduce new features, **update the relevant docs** (or create new ones) as part of the change.

Key documents:
- `docs/architecture.md` -- system overview, component model, rendering modes, client-server protocol, caching, and planned features.

## Conventions

- Source files live under `src/`.
- Components live under `src/components/` with all files colocated (`.server.js`, `.js`, `.html`, `.css`). See `docs/architecture.md` for naming conventions.
- Static/client-side framework files live under `src/static/lib/`. Files there are served under the `/js` URL path (e.g. `src/static/lib/fusewire/foo.js` is served at `/js/fusewire/foo.js`).
- Tests live under `test/` using the `*.test.js` suffix. Test files import from `../src/`.
- Config files are gitignored; use `config/*.template` for templates.
